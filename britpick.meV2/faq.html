Frequently Asked Questions

How does britpick.me work?

Britpick.me searches for common Americanisms (e.g., apartment) and suggests a British equivalent. The tool searches for whole words only. It does search for the obvious variants of words (e.g., apartment and plural apartments, for verbs like curse it will search for curse, curses, cursed, and cursing), but these variants are hard-coded, not generated, so sometimes I may have missed one.


How much fic can I paste in the box?

The short answer to this is: I don't know. I've personally pasted in a few thousand words at a time and not run into any problems. There probably is a limit, however, and it's probably a limit built into the HTML5 specifications for form elements. It's on my to-do list to find a definite answer.


What does the do in the explanations for the suggested changes?

If you click the in the explanation for a suggested change, the page will reload after rechecking your fic, excluding the word you chose to ignore (and its variants). It will ignore all instances of the word and its variants, so use this with caution. A box at the top of the page will also display the words you're currently excluding. At present, you have to go back to the Start page to start over, excluding nothing.


What do you mean, customizable?

The database behind my search script is just a simple .csv spreadsheet. Eventually I plan to add the capability to upload your own .csv spreadsheet (in the format I specify) so that you can customize your britpick.me experience.


Are there any known bugs?

Oh, probably. You can look at the project's GitHub Issues page to see what they are.


What other features are you planning to add?

  • The ability to exclude entire categories of words to search for. For example, some Americanisms such as calling a year in school a grade are related only to education as a category, and if your fic doesn't deal with the education system at all, it might be annoying (and/or confusing) to see other uses of the word grade in your fic picked out by the tool.

  • The ability to accept or reject the tool's suggestions so that you end up with text that you can copy and paste back into the document you're working from. Don't hold your breath on this one, though—it's a sizable task.


Are you British? Will you Britpick for me?

Nope, I'm not. (And therefore: no, I won't/can't.) I compiled my spreadsheet of American and British equivalents from (mainly) these two sources:

  • Brit-pick Hints for Sherlock Authors by red_adam

  • Britpicking Master Post by needlesslydefiantwithtea


You're wrong about _____!

If you think there's an error in my American/British equivalents, I would suggest first checking the sources listed above to see if the potential mistake you caught is discussed there. If it isn't (or if you still disagree), then you can either submit a bug report at the project's GitHub Issues page (requires a GitHub account, which probably doesn't apply to many of you), or you can e-mail me about it or submit an Ask (anonymously, if you like) on my Tumblr.


How can I report bugs or suggest a feature?

If you have a GitHub account, you can use the project's GitHub Issues page to report bugs or suggest features.

I've also created a Trello Board for this project. On the board you can view my to-do list and vote on planned enhancements in addition to suggesting new ones. You don't have to have a Trello account to join the board, but if you don't have an account, you'll have to give me your e-mail address so that I can add you to the board manually. I'm happy to do that if you're interested—just ask!

If none of those options appeal, you can also just shoot me an e-mail or submit an Ask (anonymously, if you like) on my Tumblr. If you're reporting a bug, please let me know what browser (Chrome, Firefox, Safari, etc.) and operating system (Windows, Mac, Linux) you're using. I use a Mac and test on Chrome and Firefox, but I haven't done more extensive testing than that.

If you have an idea for a feature, please suggest it! I want this tool to serve the community as best it can, but I can't do that if my fellow fic authors don't tell me what they want.


I want to contribute code!

If you want to contribute code, you absolutely can. Just use the standard GitHub procedures for this: fork the project, add your code, and submit a pull request. I reserve the right to decline pull requests, of course, but I'm open to collaboration.

As the GitHub README for the project explains, right now britpick.me uses only front-end web technologies (HTML, CSS, JavaScript). There are two reasons for this:

  • I can host a front-end only webapp for free on GitHub using the GitHub Project Pages feature. I am (relatively) poor. Therefore, this is pretty important.

  • I don't have much experience with back-end/server-side technologies.

The moral: if you're a back-end developer who has some spare energy and spare cash to put into this project, then (and only then) would I be willing to expand this app into more of a full-fledged webapp with a much fuller feature set.
